## 发现的问题

### BUG-ST-01 — Maya 响应延迟过高

**严重程度：** HIGH  
**表现：** Maya 平均响应时间 30~40s，P95 接近 60s，远高于 Oscar 和 Daniel（avg ~10s）  
**影响：** 用户等待体验差，高并发下容易触发 60s 网关超时  
**建议：** 排查 Maya（marketing）Agent 的 LLM 配置，检查模型选择、prompt 复杂度、向量检索耗时

### BUG-ST-02 — 服务器过载时 Maya / Oscar 大量 504

**严重程度：** HIGH  
**复现条件：** 服务器在短时间内接收大量测试请求后，后续请求全部 504  
**表现：** Kong API Gateway 强制断开超时请求，返回 504 Gateway Timeout  
**建议：**
- 增加请求队列和限速机制，防止后端积压
- 延长 Kong 超时配置（60s → 120s）
- 服务器端增加请求熔断保护

### BUG-ST-03 — 测试账号 lead_required 错误

**严重程度：** MEDIUM（产品逻辑 bug）  
**表现：** hemopiv218 和 wipepix900 账号在 dashboard 页面也触发 lead 收集拦截（HTTP 209）  
**根本原因：** 服务器未区分 `platform: "dashboard"` 和外部 widget 访客，已登录用户不应触发 lead gate  
**建议：** 后端判断 platform 字段，dashboard 用户直接跳过 lead 检查

## 建议

1. **优先排查 Maya 延迟问题**，avg 30s 是 Oscar/Daniel 的 3 倍，影响用户体验
2. **服务器端加入请求保护机制**（限流、熔断），避免高负载下全部崩溃
3. **修复 lead_required 对 dashboard 用户的误触发**（产品 bug）
4. **增加更多测试账号**，多用户并发测试目前受限于单账号 quota
